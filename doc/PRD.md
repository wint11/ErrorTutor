# 个性化课后辅导系统产品需求文档 (PRD)

## 1. 文档概述 (Document Overview)

| 项目名称 | 个性化课后辅导系统 (基于错误驱动与RAG技术) |
| :--- | :--- |
| **文档版本** | V1.0 |
| **文档状态** | 草稿 |
| **最后更新** | 2026-03-15 |

### 1.1 背景与目标
本系统旨在构建一套面向学生错误、具备学习闭环能力的智能课后辅导系统。通过综合利用大语言模型（LLM）的自然语言理解与生成能力，以及检索增强生成（RAG）技术在知识约束与可靠性方面的优势，解决传统辅导中缺乏个性化、准确性不足及无法形成学习闭环的问题。

**核心目标：**
1.  **提升准确性与权威性**：利用RAG机制，基于权威教材与题库生成讲解，减少LLM幻觉。
2.  **个性化辅导**：基于学生错误行为构建画像，动态调整讲解策略。
3.  **错误驱动闭环**：实现“错误发现—原因归因—强化训练”的完整学习闭环。
4.  **验证可行性**：验证LLM+RAG在教育场景下的协同效果。

### 1.2 适用范围
本期版本（MVP）聚焦于**初中数学一元一次方程应用题**的辅导场景。

---

## 2. 用户角色 (User Personas)

| 角色 | 描述 | 核心需求 |
| :--- | :--- | :--- |
| **学生** | 初中生，在解应用题时常遇到困难，尤其是审题、设元和列方程环节。 | 需要清晰的解题引导，而非直接给答案；希望了解自己错在哪里；需要针对性的练习巩固。 |
| **系统管理员** | 负责维护知识库、监控系统运行状态。 | 需要方便地导入题目、解析和错误案例；需要监控RAG检索的准确性。 |

---

## 3. 功能架构 (Functional Architecture)

系统核心功能划分为四大模块：

1.  **题目输入与解析模块**：负责题目的数字化与结构化。
2.  **思维引导与错误采集模块 (核心)**：分步引导学生解题，隐式采集错误数据。
3.  **错误诊断与个性化讲解模块 (核心)**：基于采集的数据诊断错误类型，结合RAG生成个性化讲解。
4.  **针对性练习推荐与学习闭环模块**：基于错误类型推荐变式题，形成学习闭环。

---

## 4. 详细功能需求 (Detailed Requirements)

### 4.1 题目输入与解析模块

#### 4.1.1 题目输入
*   **功能描述**：用户通过拍照或粘贴文本的方式输入题目。
*   **输入方式**：
    *   **拍照上传**：支持调用摄像头或相册上传图片，集成OCR识别功能。
    *   **文本粘贴**：提供文本输入框。
*   **交互规则**：
    *   系统展示识别后的文本，支持用户手动修正（容错设计）。
    *   提供[确认题目]和[重新识别]按钮。

#### 4.1.2 题目预判
*   **功能描述**：系统自动判断题目是否属于“一元一次方程应用题”。
*   **逻辑规则**：
    *   是：进入辅导流程。
    *   否：提示“当前仅支持一元一次方程应用题辅导”，并中止流程。

#### 4.1.3 题目结构化解析
*   **后台逻辑**：系统在后台将题目解析为结构化数据（题干、所求量、已知条件），用于后续的对比诊断。

---

### 4.2 思维引导与错误采集模块 (核心创新)

本模块不直接给出答案，而是通过分步引导，暴露用户的思考过程。

#### 4.2.1 步骤1：理解题意
*   **系统提问**：“这道题要我们求什么？已知条件有哪些？”
*   **用户操作**：
    *   自由输入文本。
    *   或选择预设标签（如“求数量”、“求时间”、“求总价”）。
*   **数据采集**：记录用户对题意理解的准确性（与标准解析对比）。
*   **交互控制**：提供[上一步]、[下一步]、[跳过]按钮。

#### 4.2.2 步骤2：设未知数
*   **系统提问**：“你打算设哪个量为x？为什么？”
*   **用户操作**：输入如“设买了x本书”或从选项中选择。
*   **数据采集**：记录设元是否合理、单位是否统一。

#### 4.2.3 步骤3：列方程
*   **系统提问**：“根据题意，你列出的方程是？”
*   **用户操作**：输入方程表达式（如“15x=75”）。
*   **数据采集**：记录方程的等量关系是否正确。

#### 4.2.4 步骤4：求解与验答 (可选)
*   **系统提问**：“解出来是多少？代入原题验证一下？”
*   **用户操作**：输入计算结果。
*   **数据采集**：记录计算过程与验算意识。

**通用规则**：
*   系统静默记录每步耗时、修改次数、犹豫行为。
*   用户可随时回溯修改上一步。

---

### 4.3 错误诊断与个性化讲解模块 (核心创新)

#### 4.3.1 错误诊断逻辑
系统将用户每一步输入与标准解题路径对比，判定错误类型（四选一）：

| 错误类型 | 判定规则 | 后续策略 |
| :--- | :--- | :--- |
| **审题错误** | 提取的已知条件与原文不一致，或遗漏关键信息。 | 引导重读题目，高亮关键词。 |
| **设元错误** | 未知数定义与所求不匹配，或单位混淆。 | 举例说明“设什么”的重要性。 |
| **列式错误** | 方程左右不等价，或等量关系找错。 | 核心！用生活类比解释正确关系。 |
| **计算错误** | 方程正确但求解出错。 | 跳过概念讲解，直接提示计算技巧/易错点。 |

#### 4.3.2 RAG检索逻辑 (三路检索)
在生成讲解前，系统执行三路检索：
1.  **题目级检索**：找相同/相似题目的标准解析（保证步骤权威性）。
2.  **错误级检索**：找同类错误的讲解案例（复用验证过的教学话术）。
3.  **知识点级检索**：找薄弱知识点的教材原文（确保概念一致性）。

**融合规则**：优先展示错误级检索内容，用知识点级检索内容校验，参考标准解析步骤。

#### 4.3.3 个性化讲解生成
系统根据诊断结果和检索内容，动态组织讲解：
1.  **情感反馈**：错误时给予鼓励（“这个思路很有代表性...”），正确时给予肯定。
2.  **主体内容**：
    *   审题错误 → 展示“题目关键信息提取模板”。
    *   设元错误 → 给出“设未知数三问法”。
    *   列式错误 → 引用教材原文，使用“如果...那么...”类比生活场景。
    *   计算错误 → 提供“移项变号口诀”。
3.  **总结升华**：一句话总结核心要点，给出行动指南。

**约束**：讲解内容必须引用知识库中的权威表述。

---

### 4.4 针对性练习推荐与学习闭环模块

#### 4.4.1 练习推荐策略
基于错误类型推送1道变式练习题：
*   **审题错误** → 推荐题干结构相似但关键数字/条件变化的题目。
*   **设元错误** → 推荐所求量相同但情境变化的题目（强化迁移）。
*   **列式错误** → 推荐等量关系相同但增加干扰项的题目（巩固建模）。
*   **计算错误** → 推荐方程结构相同仅数字变化的题目（降低认知负荷）。

#### 4.4.2 学习闭环
*   **错题自动归档**：记录题目、错误类型、讲解时间。
*   **知识点标签**：关联核心知识点（如“等量关系”、“移项法则”）。
*   **复习提醒**：同一知识点错误≥2次，24小时后推送复习提示。
*   **进度可视化**：文字统计各错误类型出现次数。

## 5. 知识库设计 (Knowledge Base for RAG)

### 5.1 数据结构
1.  **题目本体**：
    *   题干文本、标准答案。
    *   分步解析（每步包含操作内容+对应知识点）。
2.  **错误案例库 (核心)**：
    *   错误类型标签。
    *   典型错误作答示例。
    *   错误原因分析（教师视角）。
    *   针对性讲解话术模板。
    *   类比案例素材。
3.  **知识点索引**：
    *   知识点ID、名称、教材页码引用。
    *   前置依赖关系。

---

## 6. 界面交互原型 (UI Prototype)

### 6.1 关键页面流
1.  **题目输入页**：拍照/文本输入 -> 确认/编辑 -> 提交。
2.  **思维引导页**：
    *   顶部：题目展示。
    *   中部：当前步骤问题（如“这道题要求什么？”）。
    *   操作区：输入框/选择题。
    *   底部：导航按钮（上一步、下一步、跳过）。
3.  **诊断与讲解页**：
    *   诊断卡片：“你犯了【列式错误】，原因：...”
    *   讲解卡片：权威引用（教材P32...） + 类比解释 + 对比分析。
    *   反馈区：[听懂了] / [没明白，再讲]。
4.  **练习推荐页**：
    *   推荐理由：“针对你的列式错误，练一练...”
    *   新题目展示。
    *   [开始作答]。

---

## 7. 非功能需求 (Non-Functional Requirements)

1.  **响应速度**：OCR识别<3秒，单步引导响应<1秒，讲解生成<5秒。
2.  **准确性**：OCR识别率>95%，题目类型判断准确率>98%。
3.  **兼容性**：Web端适配主流浏览器。
4.  **数据安全**：用户上传的题目图片不涉及隐私泄露。

## 8. 验收标准 (Acceptance Criteria)

### 8.1 基础功能 (Must Have)
- [ ] 能输入/识别一道一元一次方程应用题。
- [ ] 能完成4步思维引导交互（支持跳过）。
- [ ] 能识别4种错误类型中的至少3种。
- [ ] 能生成含权威引用的个性化讲解。
- [ ] 能推荐1道针对性变式题。

### 8.2 创新功能 (Should Have)
- [ ] 讲解内容体现“鼓励式+类比式”教学策略。
- [ ] 推荐题目能说明“为什么针对你的错误”。
- [ ] 支持错题归档与简单统计展示。

### 8.3 体验功能 (Nice to Have)
- [ ] 每步提供“回溯”选项。
- [ ] 讲解支持“没明白，再讲一次”。
- [ ] 界面有清晰的状态提示。
